MQTT

Message Queuing Telemetry Transport

O MQTT é o protocolo padrão para mensagens IoT. Criado para ser extremamente leve, ele utiliza o modelo de Publicação/Assinatura (Pub/Sub), sendo ideal para conexões remotas onde a largura de banda é limitada ou o consumo de energia é crítico.

Como funciona a Arquitetura?

Diferente do HTTP (Request/Response), no MQTT os dispositivos não conversam diretamente. Tudo passa pelo Broker.

Publish/Subscribe

Um sensor (Publisher) envia dados para um "Tópico". Quem quiser o dado (Subscriber) assina esse tópico e recebe a atualização automaticamente.

O Broker (O Maestro)

É o servidor central que recebe, filtra e encaminha as mensagens para os destinatários corretos, garantindo que o sensor não precise gastar energia procurando clientes.

Aplicações Reais

1. Automação Residencial

Controle de lâmpadas, ar-condicionado e sensores de presença via plataformas como Home Assistant.

2. Monitoramento de Frotas

Rastreadores GPS enviando coordenadas para servidores centrais com baixo consumo de dados móveis.

3. Indústria 4.0

Monitoramento em tempo real de vibração e temperatura de motores em linhas de produção.

Ficha Técnica
  • Alcance: Depende da Rede (TCP/IP/Wi-Fi/4G)
  • Consumo: Extremamente Baixo
  • Taxa de Dados: Baixa (Foco em Telemetria)
  • Segurança: TLS/SSL (Porta 8883)
Níveis de QoS
  • QoS 0: No máximo uma vez (Entrega rápida).
  • QoS 1: Pelo menos uma vez (Garante entrega).
  • QoS 2: Exatamente uma vez (Crítico/Sem duplicatas).
